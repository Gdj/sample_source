nanoScroller.js

nanoScroller.js nanoScroller.js는 Mac OS X Lion 스타일의 스크롤바를 웹 사이트에 구현할 수있는 간단한 방법을 제공하는 jQuery 플러그인입니다. .nano> .nano-content 인 최소한의 HTML 마크 업을 사용합니다. 다른 스크롤 막대 div 요소 인 .pane> .nano-slider는 템플릿 처리가 복잡해지지 않도록 런타임 중에 추가됩니다. 최신 버전은 네이티브 스크롤 기능을 사용하며 iPad, iPhone 및 일부 Android 태블릿에서 작동합니다.

사용을 시작하려면 세 가지 기본 사항이 필요합니다:

1. Markup

플러그인을 작동 시키려면 다음 유형의 마크 업 구조가 필요합니다.:

<div id="about" class="nano">
	<div class="nano-content"> ... content here ...  </div>
</div>

HTML 마크 업을 복사하십시오. .nano를 콘텐츠와 관련된 것으로 변경하십시오. ID 참조가있는 상위 div가있는 경우 해당 속성을 제거 할 수도 있습니다. 예 : #parent> .nano. 나노 및 나노 컨텐츠 클래스 이름은 플러그인 옵션을 통해 사용자 정의 할 수 있습니다 (이 경우 플러그인의 CSS 파일에서도 이름을 변경해야합니다).

2. CSS

페이지의 head 섹션에있는 nanoscroller.css 파일에 링크하십시오 (... 페이지의 주요 스타일 시트 파일로 내용을 복사하십시오). 

<link rel="stylesheet" href="nanoscroller.css">

컨테이너의 너비와 높이를 지정하고 스크롤바에 사용자 정의 스타일을 적용해야합니다. 다음은 그 예입니다.:

.nano { background: #bba; width: 500px; height: 500px; }
.nano .nano-content { padding: 10px; }
.nano .nano-pane   { background: #888; }
.nano .nano-slider { background: #111; }

3. JavaScript

이 스크립트를 실행하면 nanoScroller 플러그인이 .nano className을 갖는 모든 DOM 요소에 적용됩니다.

$(".nano").nanoScroller();

Advanced methods

scroll

To scroll at the top:

$(".nano").nanoScroller({ scroll: 'top' });

To scroll at the bottom:

$(".nano").nanoScroller({ scroll: 'bottom' });

To scroll at the top with an offset value:

$(".nano").nanoScroller({ scrollTop: value });

To scroll at the bottom with an offset value:

$(".nano").nanoScroller({ scrollBottom: value });

To scroll to an element:

$(".nano").nanoScroller({ scrollTo: $('#a_node') });

stop:

작업을 중지합니다. 이 옵션은 모든 이벤트 바인딩을 비활성화하고 UI에서 가제트 스크롤바를 숨기도록 플러그인에 지시합니다.

$(".nano").nanoScroller({ stop: true });

destroy:

Destroys nanoScroller and restores browser's native scrollbar.

$(".nano").nanoScroller({ destroy: true });

flash:

플러그인 설정에서 정의 된 시간 동안 스크롤바 가젯을 플래싱합니다 (기본값은 1,5 초). 사용자를 기다리는 콘텐츠가 더 많다는 것을 사용자 (예 : 페이지로드)에 표시하려는 경우 유용합니다.

$(".nano").nanoScroller({ flash: true });

nanoScroller();

스크롤바를 새로 고침하십시오. 이것은 단순히 스크롤바 가젯의 위치와 높이를 다시 계산합니다.

$(".nano").nanoScroller();

Custom events

'scrollend'

사용자가 'scrollend'이벤트는 사용자가 content 요소의 끝까지 스크롤 할 때마다 요소에서 트리거됩니다 (사용자가 아래로 스크롤하려고 할 때 한 번 이상 트리거되지 않고 이미 스크롤 가능한 내용의 끝에 도달 한 경우).

$(".nano").bind("scrollend", function(e){
				console.log("current HTMLDivElement", e.currentTarget);
		});

일부 브라우저는 매번이 이벤트를 두 번 이상 트리거하므로 jQuery의 정상적인 .bind 또는 .on 을 사용하는 대신 사용자 정의 이벤트를 듣기 위해 이 작은 jQuery는 플러그인을 디버깅 하여 브라우저가 귀하의 기능을 100ms에 한 번 이상 실행하지 못하도록합니다.

$(".nano").debounce("scrollend", function() {
				alert("The end");
		}, 100);

'scrolltop'

Same as the 'scrollend' event, but it is triggered every time the user has scrolled to the top of the content element.

Plugin Options

There are a few options that you can change when running nanoScroller, e.g. $(".nano").nanoScroller({ paneClass: 'myclass' });

iOSNativeScrolling

iOS 5 이상에서 기본 스크롤을 사용하려면 true로 설정하십시오. 이렇게하면 iOS 5+에서 사용자 정의 nanoScroller 스크롤바가 비활성화되고 기본 nanoScroller가 대신 사용됩니다. 기본 iOS 스크롤바는 대개 훨씬 잘 작동하지만 특정 상황에서는 버그가있을 수 있습니다.

iOSNativeScrolling 옵션을 사용하면 iOS 기본 스크롤을 지원하는 장치에 .pane 및 .slider 요소가 생성되거나 추가되지 않습니다.

Default: false

$(".nano").nanoScroller({ iOSNativeScrolling: true });

sliderMinHeight

슬라이더 요소의 최소 높이를 설정합니다.

Default: 20

$(".nano").nanoScroller({ sliderMinHeight: 40 });

sliderMaxHeight

슬라이더 요소의 최대 높이를 설정합니다.

Default: null

$(".nano").nanoScroller({ sliderMaxHeight: 200 });

preventPageScrolling

content div의 상단이나 하단에 도달하면 페이지 스크롤을 방지하려면 true로 설정하십시오.

Default: false

$(".nano").nanoScroller({ preventPageScrolling: true });

disableResize

nanoscroller에서 크기 조정을 사용하지 않으려면 true로 설정하십시오. resize 이벤트를 완전하게 제어하고 싶을 때 유용합니다. 이 옵션을 true로 설정하면 스크롤이 이상한 동작을하지 않도록 reset 메소드를 호출해야합니다.

Default: false

$(".nano").nanoScroller({ disableResize: true });

alwaysVisible

true로 설정하면 자동 숨기기에서 스크롤바가 멈 춥니 다.

Default: false

$(".nano").nanoScroller({ alwaysVisible: true });

flashDelay:

이 설정을 사용하여 flash 옵션을 활성화 한 경우 스크롤 막대 숨기기 지연을 밀리 초 단위로 지정하십시오.

$(".nano").nanoScroller({ flashDelay: 1000 });

Default: 1500

paneClass

스크롤 바의 트럭 요소의 클래스 명입니다. 이 설정을 변경하면 플러그인의 CSS 파일에서 설정을 변경해야합니다.

Default: 'nano-pane'

$(".nano").nanoScroller({ paneClass: 'scrollPane' });

sliderClass

스크롤 바의 샘 요소의 클래스 명입니다. 이 설정을 변경하면 플러그인의 CSS 파일에서 설정을 변경해야합니다.

Default: 'nano-slider'

$(".nano").nanoScroller({ sliderClass: 'scrollSlider' });

contentClass

콘텐츠 div의 클래스 이름입니다. 이 설정을 변경하면 플러그인의 CSS 파일에서 설정을 변경해야합니다.

Default: 'nano-content'

$(".nano").nanoScroller({ contentClass: 'sliderContent' });

tabIndex

스크롤 가능한 내용의 탭 순서를 설정하십시오. 탭으로 이동할 때 스크롤 가능한 내용을 건너 뛰려면 -1로 설정하십시오.

Default: 0

$(".nano").nanoScroller({ tabIndex: 0 });

How it works

플러그인은 스크롤바 가젯 (스타일을 정의하기 위해 CSS가 미리 정의되어 있음)을 만든 다음 .nano-content 의 스크롤 이벤트를 구독하여 작동합니다. 마우스 누르기 및 끌기 이벤트는 .nano-pane 및 .nano-pane> .nano-slider 를 사용하여 네이티브 스크롤바의 메커니즘을 에뮬레이트 할 수 있습니다. 시스템 스크롤바는 뷰포트에서 숨겨져 있습니다 (그림 1). 이렇게함으로써 우리는 여전히 OS가 제공하는 자연스러운 스크롤링 경험을 유지합니다.

우리는 여전히 수평 스크롤 기능을 수행하고 있습니다. 프로젝트에 참여하는 데 관심이 있다면 자유롭게 포크하고 끌어 오기 요청을 만들 수 있습니다.

Development

소스에서 nanoScroller를 빌드하려면 다음 라이브러리가 설치되어 있어야합니다.:

  • Node.js and npm: homepage / download
  • Grunt: homepage | npm install -g grunt-cli

How to build & contribute

  1. Make sure that you have Grunt installed.
  2. In terminal move to nanoscroller folder and run npm install to install all dependencies.
  3. Make all Javascript changes in Coffeescript file(s), CSS changes in CSS file(s).
  4. run grunt to build nanoScroller
  5. Make sure that all changes are valid and open a pull request.

How to run tests

  1. You need to have PhantomJS installed. On Mac OS X the easiest way is to install Homebrew and run brew install phantomjs.
  2. run grunt test in terminal

Browser compatibility

Tested desktop browsers:

  • IE7+
  • Firefox 3+
  • Chrome
  • Safari 4+
  • Opera 11.60+

Mobile support:

  • iOS 5+ (iPhone, iPad and iPod Touch)
  • iOS 4 (with a polyfill)
  • Android Firefox
  • Android 2.2/2.3 native browser (with a polyfill)
  • Android Opera 11.6 (with a polyfill)
  • If you see it's broken on other tablets and mobile devices, please file a ticket in the git repo. Along with model name, and OS of the device.

If you find a bug, please report here at the issues section.

Using a polyfill for better mobile browser support

You can use overthrow.js polyfill (~1.5kb minified and gzipped) to make nanoScroller work on many mobile devices. It emulates CSS overflow (overflow: auto;/overflow: scroll;) in devices that are lacking it.

To use overthrow, link to the javascript file in your HTML document...

<script src="overthrow.js"></script>

...and add an overthrow class to your content div.

<div id="about" class="nano">
				<div class="overthrow nano-content"> ... content here ...  </div>
		</div>
